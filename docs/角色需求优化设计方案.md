# 角色需求优化设计方案

## 一、现状分析

### 1.1 现有数据模型

当前 `User` 表结构：
```prisma
model User {
  id            BigInt  @id @default(autoincrement())
  role          Int     @default(1)  // 1=需求方 2=服务方 3=平台运营
  currentRole   Int     @default(1) @map("current_role")
  // ...
}
```

**问题**：`role` 字段只能存储单一角色值，无法表示"用户同时拥有多个角色"。

### 1.2 现有角色切换逻辑

后端 `switchRole` 校验：
```typescript
// user.service.ts
async switchRole(id: number, role: number) {
  const user = await this.prisma.user.findUnique({ where: { id } });
  // 只能切换到自己已有的角色
  if (user.role !== role) {
    throw new BadRequestException('无权切换到该角色，该角色未分配');
  }
  return this.prisma.user.update({
    where: { id },
    data: { currentRole: role },
  });
}
```

**问题**：由于 `role` 字段是单值，用户实际上无法切换到其他角色。

### 1.3 现有前端实现

- "我的"页面：显示切换按钮，点击切换到另一个角色
- 角色存储：`uni.getStorageSync('currentRole')`
- 登录后：使用 `currentRole` 作为当前角色

## 二、需求梳理

### 2.1 核心需求

| 需求 | 说明 |
|------|------|
| **多角色支持** | 同一个用户可以同时是需求方和服务方 |
| **默认需求方** | 用户登录后默认以需求方身份使用小程序 |
| **角色切换** | 有多个角色的用户可在"我的"页面切换角色主页 |
| **记忆上次选择** | 切换后下次打开小程序，默认是上次选择的角色 |
| **运营配置** | 服务方角色需由运营人员在后台添加，不允许用户自行申请 |

### 2.2 用户故事

> **作为用户**
> 我登录进来默认是需求方角色
> 如果我是服务方，需要联系运营人员在后台添加服务方角色后，方可在小程序端进行服务方视角切换

---

## 三、改造方案

### 3.1 数据模型改造

#### 方案选择：JSON 数组存储

将 `User.role` 从单值改为 JSON 数组，存储用户拥有的所有角色：

```prisma
model User {
  id            BigInt   @id @default(autoincrement())
  roles         Json     @default("[1]")    // 角色数组，如 [1] 或 [1,2]，1=需求方 2=服务方 3=平台运营
  currentRole   Int      @default(1)        // 当前选中的角色
  // ...
}
```

**优点**：
- 数据库改动小，只需修改一个字段类型
- 无需新增关联表
- 查询时 JSON_CONTAINS 即可判断

**字段含义变更**：
| 字段 | 原含义 | 新含义 |
|------|--------|--------|
| `roles` | 单个角色值 | 角色数组，表示用户拥有的所有角色 |
| `currentRole` | 当前角色 | 记住用户上次选择的角色（登录时使用） |

#### 角色常量定义

```typescript
// constants.js
export const ROLE_MAP = {
  1: '需求方',
  2: '服务方',
  3: '平台运营'
}

export const ROLE_LIST = [
  { value: 1, label: '需求方' },
  { value: 2, label: '服务方' }
]
```

#### 3.2.2 角色切换接口（switchRole）

**修改点**：校验目标角色是否在用户的角色列表中

```typescript
// user.service.ts
async switchRole(id: number, role: number) {
  const user = await this.prisma.user.findUnique({ where: { id } });

  // 将 roles JSON 解析为数组
  const roles = typeof user.roles === 'string'
    ? JSON.parse(user.roles)
    : user.roles || [1];

  // 校验：目标角色是否在用户已分配的角色列表中
  if (!roles.includes(role)) {
    throw new BadRequestException('无权切换到该角色，请联系运营人员添加');
  }

  // 校验：目标角色是否为有效角色
  if (![1, 2].includes(role)) {
    throw new BadRequestException('无效的角色');
  }

  return this.prisma.user.update({
    where: { id },
    data: { currentRole: role },
  });
}
```

#### 3.2.3 新增：分配角色接口（后台管理调用）

```typescript
// admin.service.ts - 新增方法
async assignRoles(userId: number, roles: number[]) {
  // 校验角色值
  const validRoles = roles.filter(r => [1, 2, 3].includes(r));
  if (validRoles.length === 0) {
    throw new BadRequestException('无效的角色');
  }

  // 确保需求方角色始终存在（所有用户默认都是需求方）
  if (!validRoles.includes(1)) {
    validRoles.push(1);
  }

  return this.prisma.user.update({
    where: { id: userId },
    data: { roles: JSON.stringify(validRoles) },
  });
}
```

#### 3.2.4 新增：获取用户角色列表接口

```typescript
// user.service.ts - 新增方法
async getUserRoles(id: number) {
  const user = await this.prisma.user.findUnique({
    where: { id },
    select: {
      id: true,
      roles: true,
      currentRole: true,
    },
  });

  const roles = typeof user.roles === 'string'
    ? JSON.parse(user.roles)
    : user.roles || [1];

  return {
    id: user.id,
    roles,
    currentRole: user.currentRole,
    roleLabels: roles.map(r => ROLE_MAP[r] || '未知'),
  };
}
```

#### 3.2.5 API 路由汇总

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/users/profile` | 获取用户资料（新增 roles 字段） |
| POST | `/users/switch-role` | 切换当前角色（修改校验逻辑） |
| GET | `/users/roles` | 获取用户角色列表 |
| PUT | `/admin/users/:id/roles` | 分配用户角色（后台管理） |

### 3.3 前端改造

#### 3.3.1 用户状态管理

```javascript
// store/modules/user.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { getToken, setToken, removeToken, getSessionToken, setSessionToken, getUserInfo, setUserInfo, clearAuth, setCurrentRole, isLoggedIn, getNeedPhone, setNeedPhone } from '@/utils/auth'
import { userLogin, h5Login, phoneLogin, switchUserRole, bindPhoneForNewUser } from '@/api/auth'
import { getUserProfile, getUserRoles } from '@/api/user'
import { showSuccess } from '@/utils/util'

export const useUserStore = defineStore('user', () => {
  // 状态
  const token = ref(getToken() || '')
  const sessionToken = ref(getSessionToken() || '')
  const userInfo = ref(getUserInfo() || null)
  const roles = ref([1])  // 用户拥有的角色列表
  const currentRole = ref(uni.getStorageSync('currentRole') || 1) // 当前选中的角色
  const needPhone = ref(getNeedPhone() || false)

  // 计算属性
  const isLoggedInVal = computed(() => !!(token.value || sessionToken.value))
  const isDemander = computed(() => currentRole.value === 1)
  const isProvider = computed(() => currentRole.value === 2)
  const userName = computed(() => userInfo.value?.nickname || userInfo.value?.realName || '用户')
  const hasMultipleRoles = computed(() => roles.value.length > 1)  // 新增：是否有多个角色

  /**
   * 存储登录结果
   */
  function _storeLoginResult(data) {
    if (data.sessionToken) {
      sessionToken.value = data.sessionToken
      setSessionToken(data.sessionToken)
    }

    const info = {
      userId: data.userId,
      nickname: data.nickname,
      realName: data.realName,
      avatarUrl: data.avatarUrl,
      phone: data.phone,
      roles: data.roles,  // 新增
    }
    userInfo.value = info
    setUserInfo(info)

    // 优先使用服务端返回的 currentRole，其次使用本地存储
    const targetRole = data.currentRole ?? uni.getStorageSync('currentRole') ?? 1
    currentRole.value = targetRole
    setCurrentRole(targetRole)

    // 更新 roles 列表
    if (data.roles) {
      roles.value = data.roles
    }

    if (data.needPhone !== undefined) {
      needPhone.value = data.needPhone
      setNeedPhone(data.needPhone)
    }
  }

  /**
   * 获取用户角色信息
   */
  async function fetchUserRoles() {
    try {
      const res = await getUserRoles()
      const data = res.data || res
      roles.value = data.roles || [1]
      currentRole.value = data.currentRole || 1
    } catch (error) {
      console.error('获取用户角色失败:', error)
    }
  }

  /**
   * 切换角色
   */
  async function switchRole(role) {
    // 校验：目标角色是否在用户已分配的角色列表中
    if (!roles.value.includes(role)) {
      uni.showToast({
        title: '无权切换到该角色，请联系运营人员',
        icon: 'none'
      })
      return
    }

    try {
      await switchUserRole({ role })
      currentRole.value = role
      setCurrentRole(role)
      showSuccess('角色切换成功')

      if (role === 2) {
        uni.reLaunch({ url: '/provider/hall/index' })
      } else {
        uni.reLaunch({ url: '/pages/index/index' })
      }
    } catch (error) {
      console.error('切换角色失败:', error)
      throw error
    }
  }

  // ...

  return {
    token,
    sessionToken,
    userInfo,
    roles,           // 新增
    currentRole,
    needPhone,
    isLoggedIn: isLoggedInVal,
    isDemander,
    isProvider,
    hasMultipleRoles, // 新增
    userName,
    login,
    loginByWechat,
    loginByPhone,
    loginByPassword,
    bindPhoneForNewUserAccount,
    fetchUserInfo,
    fetchUserRoles,   // 新增
    switchRole,
    logout
  }
})
```

### 4.2 角色记忆逻辑

| 场景 | 行为 |
|------|------|
| 首次登录 | 默认进入需求方（currentRole=1） |
| 已有 currentRole | 使用保存的 currentRole |
| 切换角色后关闭小程序 | currentRole 已持久化，下次打开保持 |
| 退出登录 | 清除 currentRole，本地恢复默认 |

## 五、关键实现要点

### 5.1 需求方角色不可移除

- 所有用户注册时自动拥有需求方角色（roles 默认 [1]）
- 后台分配角色时，需求方角色不可取消

```javascript
// 后台管理 - 提交分配角色
if (!assignForm.roles.includes(1)) {
  ElMessage.warning('需求方角色不可移除')
  return
}
```

### 5.2 服务方角色需后台分配

- 用户无法在前端自行申请成为服务方
- 需联系运营人员在后台添加
- 切换角色时校验目标角色是否在用户的 roles 列表中

### 5.3 currentRole 持久化

- 每次切换角色时更新数据库 currentRole 字段
- 每次登录时使用数据库 currentRole（优先）或本地缓存

```javascript
// 登录时
const targetRole = data.currentRole ?? uni.getStorageSync('currentRole') ?? 1
currentRole.value = targetRole
setCurrentRole(targetRole)
```

## 六、数据库迁移脚本

```sql
-- 将 role 字段迁移到 roles 字段
UPDATE users
SET roles = JSON_ARRAY(role)
WHERE roles IS NULL OR JSON_LENGTH(roles) = 0;

-- 将 role=2 的用户同时赋予需求方角色（如果是服务方，必须也是需求方）
UPDATE users
SET roles = JSON_ARRAY(1, 2)
WHERE role = 2 AND (
  roles IS NULL OR
  JSON_LENGTH(roles) = 0 OR
  JSON_CONTAINS(roles, '2') = FALSE
);

-- 保留 role 字段但标记为废弃（可选）
-- ALTER TABLE users ADD COLUMN role_deprecated INT COMMENT '废弃字段，请使用 roles';
```
