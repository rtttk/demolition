- **team.controller.ts**: 新增路由 `GET /teams/recommend`（公开接口）

#### 前端 Admin
- **team.js (api)**: 新增 `setTeamRecommended()` API
- **team/index.vue**: 表格新增"推荐"列，按钮新增"设为推荐/取消推荐"操作

#### 前端小程序
- **index.vue**: 首页"优质团队"区域取消静态数据，改用 `getRecommendTeams` API

## 2. 修复 Prisma Include 报错

### 问题描述
`Case` 和 `Team` 模型没有定义 `team`、`company` 等关联关系，但代码中使用了 `include` 语句。

### 报错信息
```
Unknown field `team` for include statement on model `Case`
Unknown field `company` for include statement on model `Team`
```

### 修复内容

#### case.service.ts
- `getRecommend()`: 移除 include，手动查询关联数据
- `findById()`: 移除 include，手动查询关联数据

#### team.service.ts
- `getRecommendTeams()`: 移除 include，手动查询关联的 company 数据

### 修复方案
```typescript
// 手动查询关联数据
const teamIds = [...new Set(list.map((c: any) => c.teamId).filter(Boolean))];
const teams = teamIds.length > 0 ? await this.prisma.team.findMany({
  where: { id: { in: teamIds } },
  select: { id: true, name: true },
}) : [];
const teamMap = new Map<string, any>(teams.map((t: any): [string, any] => [t.id, t]));
```

## 3. 数据库配置优化

### 修改内容
- `.env` 文件：将数据库连接配置拆分为独立字段
```
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=app@12345
DB_NAME=demolition
DATABASE_URL=mysql://root:app%4012345@localhost:3306/demolition
```
- `schema.prisma`: 修改 datasource 使用 `env("DATABASE_URL")`

### 注意事项
- 密码中的 `@` 符号需要 URL 编码为 `%40`
- Prisma schema 修改后需要运行 `npx prisma generate`
